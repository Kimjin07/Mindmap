# AI 产业链知识图谱 · 项目文档

> 一句话定位：**让任何人——无论是完全的小白，还是已有基础的从业者——进入网站后都能在一张图里看懂「AI 产业链」由哪些部分组成，并且能从任意一个节点一层层往下深挖，最终落到具体的上市公司、股票代码与财务信息。**

---

## 1. 项目愿景

AI 是一条很长的产业链：从最上层人们能直接用到的「应用」，到背后的「模型」「基础设施」「芯片」，再到最底层的「能源」。
大多数人对它的理解是碎片化的——知道 ChatGPT，但不知道它跑在谁的芯片上；知道英伟达，但说不清它在产业链里到底处于哪一层。

本项目要做的，是把这条产业链做成**一张可交互、可无限下钻的知识地图**：

1. **看得懂全貌** —— 进入首页就是一张 AI 产业链全景图，五个层次一目了然。
2. **想深入就能深入** —— 每一块都可以点进去，里面有按「深度分层」组织的知识，小白看得懂、内行也有料。
3. **最终能落地到投资** —— 顺着任意一条链路往下走，终点是这一环节上的真实公司：公司介绍、股票代码、财报、估值等。知识库是前半段，**上市公司/股票信息是最终交付物**，为后续「知识 × 投资」的结合打基础。

---

## 2. 目标用户与核心理念

### 2.1 目标用户
- **完全小白**：听说过 AI，想搞懂「这玩意儿到底是怎么一回事、谁在赚钱」。
- **半懂的人**：知道一些概念（LLM、GPU），想把零散知识连成体系。
- **从业者 / 投资者**：想快速定位某个细分赛道里有哪些关键公司、谁是龙头。

### 2.2 核心理念：**分层探索（Progressive Disclosure）**
> 不管用户停在哪一层，他都能「看懂当前这一层」并且「知道还能往下走」。

同一个知识点提供多个深度层级，用户自己决定挖多深：

| 层级 | 名称 | 面向 | 内容形态 |
|------|------|------|----------|
| **L0** | 一句话 | 所有人 | 比喻 / 类比，30 字讲清「这是什么」 |
| **L1** | 通俗解释 | 小白 | 大白话 + 配图，讲清「为什么重要」 |
| **L2** | 体系知识 | 进阶 | 原理、分类、上下游关系 |
| **L3** | 深度专业 | 内行 | 技术细节、参数、行业数据、论文/报告引用 |
| **L4** | 公司与投资 | 投资者 | 该环节的上市公司、股票代码、财报、竞争格局 |

每个节点页面默认展开 L0–L1，往下逐层「展开 / 进入下一页」，做到**任意一层都能继续探索**。

## 3. 信息架构（产业链结构）

整体是一棵可下钻的树：**产业链 → 层 → 细分赛道 → 公司 → 财务/股票**。

```
AI 产业链 (首页全景图)
│
├── APPLICATIONS  应用层  —— 用户直接接触的产品
│     ├── Chatbots 对话机器人
│     ├── Digital Biology 数字生物
│     ├── Robotaxi 自动驾驶出租
│     ├── Enterprise AI Agents 企业智能体
│     ├── Science 科学计算
│     ├── Robotics 机器人
│     ├── Manufacturing 智能制造
│     └── AI Coder 代码助手
│
├── MODELS  模型层  —— 应用背后的「大脑」
│     ├── LLM / VLM / VLA / MMLLM / GPT
│     └── DM / GNN / MOE / SSM / LBM
│
├── INFRASTRUCTURE  基础设施层  ┐
├── CHIPS  芯片层               ├─ 算力与底座
└── ENERGY  能源层              ┘
      ├── GPU / CPU / DPU / NVLink / Network
      └── Storage / Cooling / Power / Grid / Fiber
```

> 注：这正是当前首页全景图已经呈现的结构（见 `app/page.tsx`）。全景图是整个产品的「地图入口」，点击任意方块或图标即进入对应节点。

### 下钻示例（一条完整链路）
```
首页
 └─ APPLICATIONS（应用层）
     └─ Chatbots（对话机器人）
         └─ 知识页：什么是对话机器人 / 技术原理 / 行业现状
             └─ 相关公司列表
                 └─ OpenAI、微软(MSFT)、Google(GOOGL)…
                     └─ 公司详情：简介 / 股票代码 / 财报 / 估值
```

## 4. 节点页面结构（每一块点进去的样子）

每个节点（无论是「层」「细分赛道」还是「公司」）都复用同一套**分层内容模板**：

```
┌─────────────────────────────────────────┐
│  面包屑：AI 产业链 / 应用层 / Chatbots      │
├─────────────────────────────────────────┤
│  [L0] 一句话：它就像……                     │
│  [L1] 通俗解释（默认展开）                  │
│  ─────────────────────────────────────   │
│  [L2] 体系知识（点击展开）                  │
│  [L3] 深度专业（点击展开）                  │
│  ─────────────────────────────────────   │
│  ▸ 上下游关系（往上/往下的相邻节点，可跳转） │
│  ▸ 相关公司（卡片列表 → 进入 L4 公司页）     │
│  ▸ 延伸阅读 / 名词卡片（hover 即看解释）     │
└─────────────────────────────────────────┘
```

设计要点：
- **永远有「下一步」**：每页底部都给出可继续点击的相邻节点 / 公司，避免「看完就到头」。
- **名词即时解释**：正文里出现的术语用可 hover 的「名词卡片」，小白不会被卡住。
- **深度可折叠**：L2/L3 默认折叠，想看的人才展开，避免吓退小白。

## 4.5 前端交互与视觉方向（已确定：地图概念）

**核心隐喻：产业链是一片可探索的「世界地图」。**
- 首页全景图 = 世界地图的总览（五个层）。
- 每个「层」= 一块**大陆 / 地图**（如 `应用层地图`）。
- 每个「细分赛道」= 大陆上的一个**地区 / 领地**（如「对话机器人」地区）。
- 地区里聚集着**居民 / 玩家** = 代表性产品与公司（如 ChatGPT、Claude、Gemini…）。

### 导航范式：点击图标 → 跳到对应大陆地图，并定位到该地区
- 在全景图点击某个图标（如 Chatbots）→ 整页跳转到该层的地图 `/map/apps`，并通过锚点 `#chatbots` 直接定位、高亮「你在这里」。
- 点击「层」的盒子（如 APPLICATIONS）→ 进入该层地图总览 `/map/apps`。
- 地图底部提供跳往其他大陆（层）的入口。

```
全景图(总览)            应用层地图 /map/apps
┌────────────┐  点击    ┌──────────┬──────────┬──────────┐
│APPLICATIONS│ ───────▶ │对话机器人 │ 数字生物 │ 自动驾驶 │
│  ◦Chatbots │          │ 你在这里  │          │          │
│  ◦Robotaxi │          │ ◦ChatGPT  │ ◦AlphaFold│ ◦Waymo   │
│  …         │          │ ◦Claude   │ …        │ …        │
└────────────┘          └──────────┴──────────┴──────────┘
                        每个地区 = 一个赛道，地区内 = 居民(产品/公司)
```

### 视觉
- 全景图与地图都用手绘金色风格；地图页加入隐约的坐标网格（像航海图 / 星图）。
- 地区用手绘金色「领地」边框；居民用「📍 图钉 + 名称 + 出品方」呈现。
- 通过 URL 锚点 `:target` 自动高亮「你在这里」，无需额外 JS。

> 备注：原先设想的「侧滑面板 + 公司详情页」方案已弃用，改为本地图方案。后续每个「居民」可再点进做公司/股票详情（与第 5 节的最终落点衔接）。

### 两套视觉（重要清晰度杠杆）
- **全景图 / 入口**：保留现有手绘金色风格——适合一眼看懂结构、激发探索欲。
- **知识内容 / 财报表格**：切换到更克制、高可读的版本（同色系，去手绘抖动、加大字号行距对比）——长文和数字用手绘风会很累。

## 5. 最终落点：上市公司 / 股票 / 财报

链路终点是「公司」节点，这是最终要交付、也是后续与股票结合的核心。公司页字段：

- **基本信息**：公司名、Logo、一句话定位、总部、官网
- **股票信息**：交易所、**股票代码 (ticker)**、市值、当前股价（接行情 API）
- **在产业链中的位置**：它属于哪些环节（一家公司可能横跨多层，如英伟达＝芯片＋基础设施）
- **业务拆解**：主营业务、AI 相关业务占比
- **财务**：营收 / 利润 / 增长（财报数据，可接 API 或人工维护）
- **竞争格局**：同赛道竞品、龙头/挑战者
- **延伸**：财报原文链接、研报、新闻

> 阶段策略：**先把知识库（L0–L3）做扎实**，公司/股票（L4）先用人工维护的静态数据跑通结构，**后期再接实时行情与财报 API**。

## 6. 数据模型（建议）

用统一的「节点(Node)」模型描述整棵树，公司是一种特殊节点。

```ts
// 产业链节点：层 / 细分赛道
interface Node {
  id: string;              // "apps", "apps.chatbots"
  parentId: string | null;
  type: "layer" | "category";
  name: string;            // 中文名
  nameEn: string;
  icon?: string;
  levels: {                // 分层内容
    l0: string;            // 一句话
    l1: string;            // 通俗
    l2?: string;           // 体系
    l3?: string;           // 深度
  };
  relatedCompanyIds: string[];
}

// 公司节点（L4）
interface Company {
  id: string;
  name: string;
  nameEn: string;
  logo?: string;
  oneLiner: string;
  nodeIds: string[];       // 它涉及的产业链环节（可多个）
  stock?: {
    exchange: string;      // "NASDAQ"
    ticker: string;        // "NVDA"
    marketCap?: number;
    // price 等实时字段后期接 API
  };
  business: string;
  financials?: { year: number; revenue: number; netIncome: number }[];
  competitors?: string[];  // company ids
  links?: { label: string; url: string }[];
}

// 名词卡片
interface Term { id: string; term: string; explain: string }
```

初期内容用本地 JSON/MDX 维护，放在 `content/` 目录；后期可迁移到 CMS 或数据库。

## 8. 技术栈

- **框架**：Next.js (App Router) + React + TypeScript
- **样式**：当前用全局 CSS + 内联 SVG（手绘金色风格），后续组件化
- **内容**：MDX / JSON，置于 `content/`（便于非工程同学写知识）
- **行情/财报数据**：预留接入层（如 Alpha Vantage / 富途 / Tushare 等），先静态后实时
- **部署**：Vercel（与 Next.js 天然契合）

## 11. 设计原则（贯穿始终）

1. **任何一层都自洽**：用户停在任何深度都能看懂当前内容，且总有「下一步」。
2. **不吓退小白，也不糊弄内行**：靠折叠/分层而不是「一刀切的难度」。
3. **一切皆可下钻，终点是公司**：每条知识链路最终都能落到真实公司与股票。
4. **内容与代码分离**：知识写在 `content/`，让非工程同学也能贡献。
5. **先结构后数据，先静态后实时**：先把可下钻的骨架和内容做对，再接行情/财报。
